OllamaClient

A .NET client library for Ollama — run large language models locally and integrate them into your C# applications with minimal boilerplate.

OllamaClient gives you full coverage of the Ollama REST API with first-class support for streaming, tool calling, structured outputs, multimodal vision models, and stateful multi-turn chat — all wired up through standard IHttpClientFactory and dependency injection.

   

Installation

Install OllamaClient via the .NET CLI:

dotnet add package OllamaClient

Or via the NuGet Package Manager Console:

Install-Package OllamaClient

You also need a running Ollama instance (defaults to http://localhost:11434/) with at least one model pulled, for example:

ollama pull llama3.2

Quick Start

Register the client through dependency injection, list available models, and stream a chat response:

using Microsoft.Extensions.DependencyInjection;
using OllamaClient;
using OllamaClient.Extensions;

var services = new ServiceCollection()
    .AddOllamaClient()          // defaults to http://localhost:11434/
    .BuildServiceProvider();

var client = services.GetRequiredService<IOllamaHttpClient>();

// List available models
var models = await client.GetModels(CancellationToken.None);

// Streaming chat
await foreach (var chunk in client.SendChat(new ChatStreamRequest
{
    Model = models.Models[0].Name,
    Messages = [new Message { Role = "user", Content = "Hello!" }]
}, CancellationToken.None))
{
    Console.Write(chunk.Message?.Content);
}

Features

Streaming chat

Every long-running endpoint exposes both a non-streaming and a streaming overload. The streaming overload returns an IAsyncEnumerable<T> you can iterate with await foreach — chunks arrive as the model produces them, with no manual NDJSON parsing required.

await foreach (var chunk in client.SendChat(new ChatStreamRequest
{
    Model = "llama3.2",
    Messages = [new Message { Role = "user", Content = "Tell me a short story." }]
}, ct))
{
    Console.Write(chunk.Message?.Content);
}

Stateful conversations

IStatefulConversationOllamaService is a higher-level service that automatically tracks message history across turns, so you can write natural multi-turn chat without managing the Messages array yourself.

var conversation = services.GetRequiredService<IStatefulConversationOllamaService>();

await conversation.SendMessage("llama3.2", "My name is Ada.", ct);
var reply = await conversation.SendMessage("llama3.2", "What is my name?", ct);
// reply remembers "Ada" — history is maintained automatically

Tool / function calling

Pass Tool definitions on a ChatRequest and OllamaClient surfaces any ToolCall emitted by the model, so you can dispatch to local handlers and feed the result back in the next turn.

var request = new ChatRequest
{
    Model = "llama3.2",
    Messages = [new Message { Role = "user", Content = "What's the weather in Sofia?" }],
    Tools =
    [
        new Tool
        {
            Type = "function",
            Function = new FunctionDefinition
            {
                Name = "get_weather",
                Description = "Get current weather for a city",
                Parameters = /* JSON Schema */
            }
        }
    ]
};

var response = await client.SendChat(request, ct);
foreach (var call in response.Message.ToolCalls ?? [])
{
    // dispatch call.Function.Name with call.Function.Arguments
}

Structured outputs

Set Format on GenerateRequest or ChatRequest to a JSON Schema object (or the literal string "json") and the model is constrained to return matching JSON — ideal for extracting typed data.

var request = new GenerateRequest
{
    Model = "llama3.2",
    Prompt = "Extract the city and country from: 'I live in Sofia, Bulgaria.'",
    Format = new
    {
        type = "object",
        properties = new
        {
            city = new { type = "string" },
            country = new { type = "string" }
        },
        required = new[] { "city", "country" }
    }
};

Multimodal / vision

For vision-capable models (e.g. llava), populate Message.Images with base-64 encoded image bytes alongside your text prompt.

var bytes = await File.ReadAllBytesAsync("photo.jpg", ct);
var request = new ChatRequest
{
    Model = "llava",
    Messages =
    [
        new Message
        {
            Role = "user",
            Content = "What is in this image?",
            Images = [Convert.ToBase64String(bytes)]
        }
    ]
};

Embeddings

Use Embed(EmbedRequest) against the modern /api/embed endpoint — it supports batch input and returns one vector per item.

var response = await client.Embed(new EmbedRequest
{
    Model = "nomic-embed-text",
    Input = ["First chunk of text", "Second chunk of text"]
}, ct);

// response.Embeddings[i] is the vector for input[i]

The legacy GetEmbeddings(EmbeddingsRequest) wrapper around /api/embeddings is still present but marked [Obsolete] — migrate to Embed.

Configuration

Default endpoint

The plain AddOllamaClient() registration connects to http://localhost:11434/:

services.AddOllamaClient();

Custom base URL

Point the client at a remote or non-default Ollama host:

services.AddOllamaClient(cfg =>
{
    cfg.OllamaEndpoint = "http://my-ollama-host:11434/";
});

Timeout

OllamaClient is built on top of IHttpClientFactory, so you can tune the underlying HttpClient using the standard IHttpClientBuilder — useful when pulling large models or running long generations:

services.AddOllamaClient();
services.AddHttpClient<OllamaHttpClient>()
        .ConfigureHttpClient(c => c.Timeout = TimeSpan.FromMinutes(5));

When OllamaClient, when not

Good fit

  • .NET applications that want to talk to a local or self-hosted Ollama instance with no extra abstractions
  • On-premise, air-gapped, or privacy-sensitive deployments where data must not leave the network
  • Teams already on the modern .NET stack — DI, IHttpClientFactory, IAsyncEnumerable
  • Apps that need full Ollama feature coverage: streaming, tool calling, structured JSON, vision, embeddings

Not a good fit

  • Applications that target hosted-only providers (OpenAI, Anthropic, Google) — use the relevant SDK instead
  • Workloads that need provider-agnostic LLM abstractions out of the box — consider a higher-level orchestration layer on top
  • Environments where you cannot run Ollama (or a compatible server) anywhere reachable from the app

Version 2.0.0 (31 May, 2026)

  • Added Multi-target support for net9.0 and net10.0; net8.0 support dropped.
  • Added IStatefulConversationOllamaService — higher-level service that automatically manages conversation history for multi-turn chat sessions.
  • Added Tool / function calling support: Tool, FunctionDefinition, FunctionCall, and ToolCall models; ChatRequest.Tools and ChatRequest.ToolChoice properties.
  • Added Structured outputs — the Format property on GenerateRequest and ChatRequest now accepts a JSON Schema object in addition to the literal "json" string.
  • Added Multimodal / vision support — Message.Images accepts a list of base-64-encoded image strings for vision models.
  • Added /api/embed endpoint via Embed(EmbedRequest) — supports batch embedding, the modern alternative to /api/embeddings.
  • Added think parameter on generate/chat requests and Thinking field on responses for reasoning models; GenerateRequest.Suffix for fill-in-the-middle generation.
  • Added GET /api/version → GetVersion() and GET /api/ps → GetRunningModels().
  • Added XML documentation on all public types and members; documentation XML file emitted in release builds.
  • Added Seven runnable sample projects: BasicChat, StreamingChat, ToolCalling, StructuredOutput, Multimodal, Embeddings, MiniRag.
  • Updated All NuGet dependencies bumped to latest stable; GitHub Actions matrix expanded over .NET 9 and 10 with added format-check and pack steps.
  • Updated GenerateRequest.Context type corrected from string to int[] to match the Ollama API spec.
  • Deprecated GetEmbeddings(EmbeddingsRequest) / GetEmbeddingsAsync(EmbeddingsRequest) — migrate to Embed(EmbedRequest) which uses the newer /api/embed endpoint.
  • Fixed DELETE /api/delete now uses HTTP DELETE instead of POST, matching the Ollama API spec.
  • Fixed NuGet package icon inclusion changed to None Remove + None Include with empty PackagePath, fixing NU5046 pack error under .NET 10 SDK.

Version 1.0.0 (22 April, 2024)

Initial public release — IOllamaHttpClient with full Ollama REST API coverage: generate, chat (streaming and non-streaming), create model, copy, delete, show, pull, push, embeddings, list models. DI integration via AddOllamaClient() extension method. Configurable base URL via OllamaConfiguration.
